## 1. Именование спецификаций

### Формат
```
SNN_<snake_case_название>.md
```

### Правила
- `NN` — двузначный номер (01, 02, 03, ..., 99)
- Нумерация **хронологическая**
- Название в `snake_case` (латиница, строчные буквы, подчеркивания)
- Файл размещается в `docs/2. specifications/`

### Примеры
```
S01_telegram_mcp_bridge.md
S02_search_optimization.md
```

### Статусы спецификаций

- **draft** — черновик, в разработке
- **active** — действующая, по ней ведется работа
- **done** — завершенная, все задачи выполнены
- **deprecated** — устаревшая, заменена другой спецификацией

Статус указывается в шапке документа:
```markdown
**Статус:** active
```

При завершении спецификации:
1. Изменить статус на `done` и добавить дату завершения
2. Опционально: добавить суффикс `_done` к имени файла

## 2. Именование задач

### Формат
```
TNNN_sNN_<snake_case_название>.md
```

### Правила
- `NNN` — трёхзначный номер (001, 002, ..., 999)
- `sNN` — номер спецификации, к которой относится задача
- Нумерация **сквозная** по всем задачам (в порядке создания)
- Название в `snake_case` (латиница, строчные буквы, подчеркивания)
- **Обязательно** в начале каждой задачи — ссылка на спецификацию

### Примеры
```
T001_s01_project_scaffold.md
T005_s01_embedding_worker.md
```

## 3. Определение следующего номера задачи

### Через командную строку:
```bash
find "docs/3. tasks" -name "T*.md" | grep -oP 'T\K\d+' | sort -n | tail -1
```

### Для AI-ассистентов
- При создании новой задачи **всегда** искать максимальный номер среди всех файлов в `docs/3. tasks/` (включая `Done/`)
- Проверять на конфликты: убедиться, что выбранный номер не используется

## 4. Размещение задач

### Текущие задачи (в работе)
```
docs/3. tasks/SNN_<имя>/TNNN_sNN_<snake_case>.md
```

### Завершенные задачи
```
docs/3. tasks/SNN_<имя>/Done/TNNN_sNN_<snake_case>_done.md
```

### Отменённые задачи
```
docs/3. tasks/SNN_<имя>/Done/TNNN_sNN_<snake_case>_cancelled.md
```

**При отмене** — внутри файла указать причину и дату:
```markdown
**Статус:** ОТМЕНЕНО
**Дата отмены:** ...
**Причина:** ...
```

## 5. Единый бэклог

### Расположение
```
docs/6. backlog/
```

Папка создаётся при появлении первой идеи для бэклога.

### Активация из бэклога

**Спецификации:**
1. Переместить в `docs/2. specifications/SNN_<имя>.md` (назначить номер)
2. Обновить ссылки во всех документах
3. Удалить исходник из `docs/6. backlog/`

---

## 6. Процедура переноса задачи

1. **Перенести файл** в `Done/` с суффиксом `_done.md`
2. **Добавить первой строкой** ссылку на спецификацию
3. **Удалить исходный файл** из активных задач
4. **Обновить ссылки** во всём репозитории
5. **Проверка:** grep по старому пути — результатов быть не должно

---

## 7. Правила для AI-ассистентов

### При создании новой задачи:
1. Искать максимальный номер `TNNN` среди всех файлов в `docs/3. tasks/` (включая `Done/`)
2. Проверять на конфликты
3. Добавлять префикс `` ко всем создаваемым документам

### При завершении задачи:
1. Перенести файл в `Done/` с суффиксом `_done.md`
2. Добавить ссылку на спецификацию первой строкой
3. Удалить исходный файл из активных задач
4. Обновить ссылки во всех документах

### При создании спецификации:
1. Назначить следующий номер `SNN`
2. Добавить префикс ``
3. Указать статус и дату создания
