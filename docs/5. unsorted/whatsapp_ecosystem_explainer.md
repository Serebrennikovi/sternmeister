## Часть 1: Что делает наш сервис

### Простыми словами

**Наш сервис — это "умный робот", который:**

1. **Слушает Kommo:** "Клиент перешёл в новый этап"
2. **Проверяет время:** Сейчас с 9 до 21? Если нет → откладывает до утра
3. **Берёт данные из Kommo:** Имя клиента, телефон, дата термина
4. **Формирует сообщение:** Подставляет данные в шаблон
5. **Отправляет через Wazzup24:** Используя одобренный WABA-шаблон
6. **Запоминает в базе данных:** Кому отправили, когда, сколько попыток
7. **Каждый час проверяет:** "Прошло 24ч? Клиент не ответил? → Отправить снова"
8. **Максимум 3 попытки:** 1 первая + 2 повтора. Потом прекращает.
9. **Алерт в Telegram:** Если что-то пошло не так (ошибка, нет номера и т.д.)

## Часть 2: Главное правило WhatsApp Business API

### Только одобренные шаблоны

**Нельзя просто взять и написать произвольный текст клиенту.**

Meta (владелец WhatsApp) требует:
1. Создать шаблон сообщения (текст с переменными)
2. Отправить на модерацию
3. Дождаться одобрения (24-48 часов)
4. Только после одобрения — можно отправлять

**Пример шаблона:**
```
Здравствуйте. Это {{1}}. Напоминаем о {{2}} в {{3}}. Скажите, всё в силе?
```

Где:
- {{1}} = название компании ("SternMeister")
- {{2}} = тип встречи ("термине", "консультации")
- {{3}} = дата и время ("25.02 в 14:00")

**Исключение:** Если клиент сам написал вам, у вас есть 24 часа, чтобы ответить произвольным текстом.

### Способы отправки через WhatsApp

### Вариант 1: Green API (неофициальный)

**Как работает:**
- Подключается к WhatsApp Web (как будто вы открыли WhatsApp в браузере)
- Отправляет сообщения "от лица" обычного аккаунта

**Плюсы:**
- Можно писать любой текст (без шаблонов и модерации)
- Быстро настроить

**Минусы:**
- **Неофициальный способ** — Meta может заблокировать аккаунт
- Нестабильно (если сессия "упала", нужно заново сканировать QR-код)
- Риск потерять доступ к номеру

**Наше решение:** Изначально планировали, но клиент сообщил, что у них уже есть Wazzup24 WABA. Выбрали официальный путь — надёжнее для продакшна.

---

### Вариант 2: Wazzup24 (официальный WABA, наш выбор)

**Как работает:**
- Официальный провайдер WhatsApp Business API
- Wazzup24 — посредник между нами и Meta
- Номер +49 3046690188 подключён через них

**Плюсы:**
- Официальный, стабильный
- Нет риска блокировки
- Надёжная доставка

**Минусы:**
- Только одобренные шаблоны (модерация 24-48ч)
- Нельзя изменить текст "на лету" (нужна новая модерация)

**Наше решение:** Используем. У нас 9 одобренных шаблонов.

## Часть 3: Про шаблоны (важный момент)

### Как работает модерация шаблонов

1. **Создаём шаблон в интерфейсе Wazzup24:**
   - Пишем текст: "Здравствуйте. Это {{1}}. Напоминаем о {{2}} в {{3}}. Скажите, всё в силе?"
   - Указываем категорию (например, "Reminder")
   - Добавляем кнопки (опционально): "Да, буду вовремя" / "Нет, не могу прийти"

2. **Отправляем на модерацию Meta:**
   - Wazzup24 пересылает шаблон в Meta
   - Модераторы проверяют: нет ли спама, обмана, запрещённого контента
   - Ответ приходит через 24-48 часов

3. **Одобрение:**
   - Если одобрено → получаем `templateId` (уникальный ID шаблона)
   - Используем этот ID в API для отправки

4. **Использование:**
   - В коде указываем: `templateId: "38194e93-e926-4826-babe-19032e0bd74c"`
   - Подставляем переменные: `["SternMeister", "термине", "25.02 в 14:00"]`
   - Отправляем → клиент получает готовое сообщение

### Новые сценарии (расширение — S02)

Поступил запрос на **новые автоматические сообщения**:
- За 7 дней до термина: напоминание + чеклист документов
- За 3 дня до термина: последнее напоминание
- За 1 день до термина: "не забудьте паспорт"
- В день термина: мотивационное сообщение
- После термина: запрос обратной связи

**Решение:**
Каждое новое сообщение требует **создания отдельного шаблона** и отправки на модерацию Meta (24-48ч). "Универсальный шаблон 4" НЕ подходит — у него фиксированный текст, который нельзя изменить.

**План:**
1. Написать тексты для каждого сценария (с учётом требований Meta)
2. Создать 5 новых шаблонов в Wazzup24
3. Отправить на модерацию (ждать одобрения 24-48ч)
4. После одобрения — доработать сервис под новые сценарии (триггеры по датам, а не по этапам)

## Часть 4: Почему Wazzup24, а не Green API

### Сравнение способов отправки

| Подход | Плюсы | Минусы | Наш выбор |
|--------|-------|--------|-----------|
| **Green API** | ✅ Произвольный текст<br>✅ Без модерации<br>✅ Быстро настроить | ❌ Риск бана аккаунта<br>❌ Нестабильно<br>❌ Неофициально | ❌ Не используем |
| **Wazzup24 WABA** | ✅ Официально<br>✅ Стабильно<br>✅ Нет риска блокировки | ❌ Только шаблоны<br>❌ Модерация 24-48ч<br>❌ Нельзя изменить текст быстро | ✅ **Используем** |

**Вывод:** Wazzup24 — официальный канал, безопасный для бизнеса. Green API — временное решение с риском потери номера.

### Что делает наш сервис поверх Wazzup24

Наш сервис — это **не замена** Wazzup24, а **автоматизация** поверх него:

✅ **Расписание:** Отправка только 9-21 (Wazzup24 отправляет всегда)
✅ **Автоповторы:** Через 24ч если нет ответа (Wazzup24 этого не делает)
✅ **Логика:** Максимум 3 попытки, отложенные сообщения (Wazzup24 просто отправляет)
✅ **Интеграция с Kommo:** Webhook → проверка времени → отправка → запись в CRM (Wazzup24 не знает про Kommo)

**Простыми словами:**
- **Wazzup24** — это "почтальон" (доставляет сообщения в WhatsApp)
- **Наш сервис** — это "менеджер" (решает, когда и кому отправить, сколько раз повторить)

## Часть 5: Технические детали нашего сервиса

### Технические детали (упрощённо)

**Где работает:**
- Сервер в Hetzner (Германия): 65.108.154.202
- Docker-контейнер (программа запущена в изолированном окружении)
- Доступ через ngrok tunnel: `https://shternmeister.ngrok.pro`

**Что внутри:**
- Python + FastAPI (веб-сервер, принимает webhook от Kommo)
- SQLite (база данных для хранения истории отправок)
- Cron (задача, которая запускается каждый час и проверяет повторы)
- Wazzup24 API (отправка сообщений)
- Telegram Bot (алерты при ошибках)

**Как работает:**
1. Kommo → webhook → наш сервер
2. Сервер → проверка времени → отправка или отложить
3. Сервер → Wazzup24 API → WhatsApp
4. Сервер → SQLite (запись в базу)
5. Cron каждый час → проверка "прошло 24ч?" → повтор
6. Telegram → алерт, если ошибка

### Что это даёт бизнесу

✅ **Автоматизация:** Менеджеры не тратят время на ручную отправку
✅ **Контроль:** Не беспокоим клиентов ночью (только 9-21)
✅ **Упорство:** Автоматически напоминаем, если клиент не ответил
✅ **Надёжность:** Официальный WABA, нет риска бана
✅ **Прозрачность:** Вся история в SQLite + примечания в Kommo
✅ **Мониторинг:** Telegram-алерт, если что-то сломалось
